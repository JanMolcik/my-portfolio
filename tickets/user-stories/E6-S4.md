## Goal
- As platform, I want writing publish events to revalidate detail, index, pagination, and sitemap caches, so that listing pages do not stay stale after Storyblok changes.

## Scope
- Writing webhook revalidation scope; supports BW-005.
- Delivery aligned with G-B execution group.

## Non-goals
- Unrelated module rewrites.

## Acceptance
- `writing/*` publish, unpublish, and delete events revalidate the affected detail route, `/writing`, `/sitemap.xml`, and the `/writing/page/[page]` route pattern with integration coverage.
